实习小“总结”：xlua插件学习

yuuki · 2020-7-25 · 次阅读

Lua

其实到现在也不用Lua能做什么。可能这就是刚入门吧。其实应该说是怎么用吧，互相来回调用有时候也挺麻烦的。

XLua

下载XLua后，直接复制Assets文件夹到工程目录。

开头using XLua;，就可以使用XLua的东西了。

XLua主要是靠一个LuaEnv来完成操作，首先肯定要new一个出来。

LuaEnv luaEnv = new LuaEnv();

执行lua字符串

使用DoString方法，参数只要是符合lua语法的字符串就可以。

luaEnv.DoString("print('hello world')")

Lua文件加载

使用DoString方法，执行Lua的require函数。

luaEnv.DoString("require 'fileName'");

使用这个方法，会调用一个个的loader去加载，有一个成功就终止，是被会报文件找不到。同时现在还添加了从Resource下加载，但是因为Resource只支持有限后缀，因此想把Lua脚本放下Resource下的话，需要加上txt后缀。加txt的操作只是在需要放在Resource下的时候使用。

自定义Loader加载

xlua支持自定义loader加载，来解决需要解压或加在特定文件夹下的lua文件。这个形式是固定的。

public delegate byte[] CustomLoader(ref string filepath);
public void LuaEnv.AddLoader(CustomLoader rloade)

其实那个委托可以直接定义一个函数，然后AddLoader直接传函数名也可以。（对，我就是不懂C#）定义的函数里就可以做需要的操作。需要先AddLoader之后再调用DoString去require，顺序很重要。

C#访问Lua

要从Lua映射到C#的类、委托、interface等最好打上[CSharpCallLua]属性。

获取全局的基本数据类型

通过LuaEnv.Global的Get<>方法就可以返回需要的类型，模板写类型，方法参数里写要找的变量名。比如lua中这么写。

str = "Test str";
num = 200;

C#中就可以这么访问。

string luaDtr = luaEnv.Global.Get<string>("str");
int luaNum = luaEnv.Global.Get<int>("num");

获取全局的table

Lua的table比较万能，能存很多种类的变量。一种方法是映射到一个class或者一个struct上。
比如有一个class定义为：

public class space
{
    public string str;
    public int num;
};

想要得到lua中的table，那么只能取到这样的table

table = {str = "NewStr" ,num = 500}

还可以映射到一个interface上面。
如这样一个interface

[CSharpCallLua]
public interface MappingTest
{
    int num { get; set; }
    string str { get; set; }
    string addStr(string str1,string str2);
}

就对应这样一个lua的table

MappingTest ={
    num =100,
    str = "This is a table",
    addStr = function (self,a,b)
        print("In MappingTest add function");
        return a .. b;
    end;
}

如果lua中的table中所有的key和value的类型都是一直的话，可以使用List<>或Dictionary<>来映射。
如果lua中有这样的字符串

List ={"aaa","bbb","ccc"};

C#中就可以这么取

List<string> list = luaEnv.Global.Get<List<string>>("List");

最后就是可以映射到LuaTable类中，但是官方说比较慢，比映射到接口慢一个数量级，也就没细看了。

访问全局function

一种是使用delegate，delegate声明对于function的每一个参数就声明一个输入类型的参数。多返回值的处理上，从左往右映射C#的输出参数，参数包括返回值，ref参数和out参数。使用上只要映射完成后就当普通函数使用。

首先声明一个返回值为string类型，输入参数为str，输出参数为str1和str2的委托。接下来

[CSharpCallLua]
public delegate string TestDelegate(string str,out string str1,out string str2);

那么Lua方面，就对应一个参数就是一个str，但返回值需要三个。

function MultiReturn(str)
    str = str .."Added By Lua";
    return str, "This Str is From Lua","And This ONE";
end;

而在C#方面，调用可以先声明两个string类型的变量。然后传入委托的方法中即可。

TestDelegate func = luaEnv.Global.Get<TestDelegate>("MultiReturn");
string str1 ="",str2="";
string str = func("Create By Unity",out str1,out str2);

这样，C#中str会被附上返回值，也就是lua中的str。而str1和str2会从左到右被依次被赋值lua中的返回值。

还有一种方法就是使用interface。首先C#中定义一个interface并声明两个方法。

[CSharpCallLua]
public interface Test
{
    void ChangeColor(Renderer renderer);
    void CreateCylinder();
}

lua中首先需要声明一个变量，然后按照 function 变量名:方法名（参数）的格式来书写要实现的函数。

UnityCallLua = {};
function UnityCallLua:ChangeColor(renderer)
    renderer.material.color = CS.UnityEngine.Color.red;
end

function UnityCallLua:CreateCylinder()
    CS.UnityEngine.GameObject.CreatePrimitive(CS.UnityEngine.PrimitiveType.Cylinder);
end;

这样只需要Get接口类型，就可以调用了。

Test UnityCallLua;
UnityCallLua = luaEnv.Global.Get<Test>("Unitys");
UnityCallLua.ChangeColor(renderer);
UnityCallLua.CreateCylinder();

最后是映射到LuaFunction，也是性能慢一点，就没看。

Lua调用C

Lua调用C#时也最好使用[LuaCallCSharp]标签。

Lua中访问C#使用CS.namespace.class的形式访问C#类。如创建一个对象可以

newObject = CS.UnityEngine.GameObject();

对应C#中的

GameObject newObject = new UnityEngine.GameObject();

而且xlua支持重载，调用重载方法就和再C#中一样写就好。

访问静态方法、属性

Lua访问C#的静态方法直接使用CS.namespace.class.var或CS.namespace.class.function

time = CS.UnityEngine.Time.deltaTime;
CS.UnityEngine.GameObject.Find('Button');

CS.UnityEngine.Time.timeScale = 2;

如果有需要经常访问的类，可以用局部变量引用后访问。

local GameObject = CS.UnityEngine.GameObject
GameObject.Find('helloworld')

访问成员方法和属性

访问成员方法是需要先引用变量，调用构造函数即可完成。

然后使用变量进行.运算符访问成员属性，使用冒号语法糖调用成员方法。
假设有C#Class

[LuaCallCSharp]
public class CustomClass 
{
    public static string str = "call by lua";
    public static void Func(string str)
    {
        Debug.LogFormat("{0}",str);
    }
    public int num = 100;
    public int add(int a,int b)
    {
        return a + b;
    }
}

lua中就可以先实例化一个变量，然后调用就好。

customClass = CS.CustomClass();
print(CS.CustomClass.str);
print(customClass.num);
print(customClass:add(12,25));
print(CS.CustomClass.Func("This is Print By Lua"));

xlua还支持通过派生类实例访问基类的成员和方法，通过派生类访问静态方法和属性。

多参数的输入输出

如C#有这样的方法

    public string MultiReturned(string str1,string str2,ref string change,out string ostr1,out string ostr2)
    {
        change = str1 + str2;
        ostr1 = str1;
        ostr2 = str2;
        return "Down";
    }

首先输入参数有str1，str2和change。输出参数有返回值的string，ostr1和ostr2。那么lua中就需要三个返回值和三个参数。

returned,reff,out1,out2 = customClass:MultiReturned("This Print is"," Added By C#");

delegate的使用

调用delegate其实和调用普通函数一样，而其中主要是+操作符和-操作符。其中+操作符是可以把两个委托串成一个委托链，-操作符是可以从一个委托链中删除一个委托。如C#中定义这样的委托。

 public delegate void DString(string str);
    public DString ShowStr = (string str) =>
    {
        Debug.Log("This is ShowStr showed" + str);
    };
    public DString AddStr = (string str) =>
    {
        Debug.Log("This is AddStr,Added" + str);
    };

那么lua就可以这么来调用和串链。

local delegate1 = customClass.ShowStr;
delegate1("This is From Lua");
delegateline = delegate1 + customClass.AddStr;
delegateline("Lua call DelegateLine");
delegateline = delegateline - delegate1;
delegateline("Delete a delegate");

Hotfix热更新

需要进行热更新的类，方法，需要打上[Hotfix]标签。

前置操作

首先需要打开特性，添加HOTFIX_ENABLE宏（在Unity3D的File->Build Setting->Scripting Define Symbols下添加）。编辑器、各手机平台这个宏要分别设置！如果是自动化打包，要注意在代码里头用API设置的宏是不生效的，需要在编辑器设置。

执行XLua/Generate Code菜单。

注入，构建手机包这个步骤会在构建时自动进行，编辑器下开发补丁需要手动执行”XLua/Hotfix Inject In Editor”菜单。打印“hotfix inject finish!”或者“had injected!”才算成功，否则会打印错误信息。

约束

不支持静态构造函数。

目前只支持Assets下代码的热补丁，不支持引擎，c#系统库的热补丁。

API

xlua.hotfix(class, [method_name], fix)

用于注入lua补丁

  • class表示C#类，两种表示方法，CS.Namespace.TypeName或者字符串方式”Namespace.TypeName”，字符串格式和C#的Type.GetType要求一致，如果是内嵌类型（Nested Type）是非Public类型的话，只能用字符串方式表示”Namespace.TypeName+NestedTypeName”
  • method_name，方法名，可选
  • fix，如果传了method_name，fix将会是一个function，否则通过table提供一组函数。table的组织按key是method_name，value是function的方式。
base(csobj):func();

用于子类override了父类函数而要调用父类函数时用。

util.hotfix_ex(class, method_name, fix)

xlua.hotfix的增强版本，可以在fix函数里头执行原来的函数，缺点是fix的执行会略慢。

使用

[Hotfix]
public class LuaHotfix : MonoBehaviour
{
    LuaEnv luaEnv = new LuaEnv();
    public int def = 100;
    int atk { get; set; }
    const float loopTime = 5;
    float addTime = 0;
    string str { get; set; }
    Origins origins { get; set; }
    void Start()
    {
        luaEnv.AddLoader(Loader);
        luaEnv.DoString("require 'HotFixLua'");
    }
    public string add(string str1,string str2)
    {
        return str1 + str2;
    }
}

关于修改方法很简单。如想修改add函数

xlua.hotfix(CS.LuaHotfix,'add',function
(self,a,b)
     return a .. b .. "Fixed";
end)

如果要修改的函数是构造函数，对应的函数名是.ctor。造函数的热补丁并不是替换，而是执行原有逻辑后调用lua。

如果要访问私有属性需要加上

xlua.private_accessible(CS.LuaHotfix);

如果想修改属性，需要设置getter和setter。通过修改getter方法来修改。

xlua.hotfix(CS.LuaHotfix,'get_atk',function ( ... )
    return 200;
end)
print(obj.atk);

xlua.hotfix(CS.LuaHotfix,'get_str',function ( ... )
    return "Fixed";
end)

xlua.hotfix(CS.LuaHotfix,'get_origins',function ( ... )
    return {name = "FixedName",num = 250,time = 25.5};
end)

杂谈

看了下xlua热更新的原理，Unity会编译生成一个中间语言（IL），然后把IL丢到Mono虚拟机里跑。lua似乎是改CIL来注入lua脚本或更改执行路径，来达到修改函数、变量的目的。也就是说，只要是产生中间语言的都可以这么做咯？？不是很了解，而且如果C++要做这种的，难道要通过汇编来改吗？？感觉很神奇。
