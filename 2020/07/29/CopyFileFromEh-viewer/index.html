从sd卡上拷贝EhViewer下载的本子

yuuki · 2020-7-29 · 次阅读

原因

为啥干这种事情呢？因为EhViewer的下载标签实际上是不会新建文件夹的。每次还想按自己的标签弄出来本子，4400多个文件夹找的是真的累，所以就自己想办法把他从SD卡里复制出来。顺便Blog的封面也方便找了。

干活了！

首先是找下载标签，很幸运EhViewer可以导出一个数据库文件(.db)，里面有很多东西，Label，Title，GID，上传者，等都有,那么只需要Select标签下的所有GID，再根据GID去找文件夹就好了，然后就是复制！

SQLite

准备

EhViewer用的数据库是SQLite,所以只要用SQLite提供的API去读取就好了。首先需要去https://www.sqlite.org/download.html下载两个东西，一个是源码，一个是已经编译好的文件。
预编译好的压缩包里有sqlite3.def和sqlite3.dll这两个文件，我们需要把def变成lib使用。首先到VS的安装目录下面D:\Microsoft VisualStudio\VC\Tools\MSVC\14.16.27023\bin\下，x86和x64看需要，将这几个文件复制到一个文件夹。

再把刚刚的def文件复制进来。打开CMD，cd到文件夹里。输入命令LIB /DEF:sqlite3.def /machine:IX86，就可以生成lib文件了，

用VS新建工程，把lib和dll和源码里的sqlite3.h都复制到工程下，记得添加库的目录到工程里和使用#pragma comment(lib,"sqlite3.lib")。

准备差不多就到这里了。

读取

打开数据库函数为sqlite3_open(),参数第一个是数据库文件（.db）的路径，第二个参数是数据库指针的指针（**db）。数据库类型是sqlite3,可以这么打开。

sqlite3 *db;
if (sqlite3_open(dbPath.c_str(), &db))
{
    std::cout << "Open DB error" << std::endl;
    sqlite3_close(db);
    exit(1);
}

打开后就可以增删改查了，由于我只需要查，所以之说查。
查找首先需要SQL语句。SQLite使用sqlite3_stmt类型作为媒介，需要一个这个类型的指针。然后使用sqlite3_prepare_v2()函数去准备语句，参数有五个，

  • 第一个是数据库指针，
  • 第二个是UTF8编码的c风格的sql语句的字符串，
  • 第三个是SQL语句的大小（bytes），
  • 第四个是sqlite3_stmt类型的指针的指针，
  • 最后一个是解析过的string的结尾，一般填NULL就行。（对，我也不知道这个是干啥的）
sqlite3_stmt *stmt = nullptr;
sqlite3_prepare_v2(db, findTitleSql.c_str(), findTitleSql.size(), &stmt, 0);

因为我查询的是个大表，所以使用sqlite3_get_table()函数来查询，有6个参数。

  • 第一个是数据库指针
  • 第二个是UTF8编码的c风格的sql语句的字符串。
  • 第三个是结果表，使用char**储存。
  • 第四个是结果表行数 int
  • 第五个是结果表列数 int
  • 第六个是错误信息 char*
sqlite3_get_table(db, findLabelSql.c_str(), &res,&row,&col,&msg);

返回的查询结果表是char** 类型就是张二维表，第一行是；列名，剩下的是查询结果，可以根据列行的一个二重循环便利表格。

读完表应该使用sqlite3_free_table()释放表内存。

void sqlite3_free_table(char **result);

如果需要多次查表，可以准备多个语句，也可以通过sqlite3_finalize()函数释放语句,然后重新准备。

int sqlite3_finalize(sqlite3_stmt *pStmt);

获取路径

在找到了要复制的本子的GID后，需要搜索所有本子了路径，然后挑出来我们想要的路径。

因为EhViewer的目录结构很简单，download下面就是所有本子的文件夹，文件夹里只有img文件。虽然我弄了个可以递归查询目录，但是不影响。

核心函数FindFirstFile()、FindNextFile()和一个结构体WIN32_FIND_DATA。

HANDLE WINAPI FindFirstFile(
   __in          LPCTSTR lpFileName,
   __out         LPWIN32_FIND_DATA lpFindFileData
 );

 BOOL WINAPI FindNextFile(
   __in          HANDLE hFindFile,
   __out         LPWIN32_FIND_DATA lpFindFileData
 );

 typedef struct _WIN32_FIND_DATA {  
       DWORD dwFileAttributes;  
       FILETIME ftCreationTime;  
       FILETIME ftLastAccessTime;  
       FILETIME ftLastWriteTime; 
       DWORD nFileSizeHigh; 
       DWORD nFileSizeLow;  
       DWORD dwReserved0;  
       DWORD dwReserved1; 
       TCHAR cFileName[MAX_PATH];
        TCHAR cAlternateFileName[14];
 }WIN32_FIND_DATA,*PWIN32_FIND_DATA,*LPWIN32_FIND_DATA;

FindFirstFile负责找出输入路径下的第一个文件，所以给到download就好了。找到的信息会放进FindData里，找不到返回值HANDLE会为INVALID_HANDLE_VALUE。通过FindData的dwFileAttributes属性可以判断是否是文件夹。是文件夹我们就存起来，不是我们就跳过。然后要查找的路径需要添加一个.的占位符，必须要有，否则会找不到文件。当然也可以在这里就判断是不是我们想要的，是就把文件也找出来，不是就跳过。个人只是想把函数功能尽可能单一，所以就没这么做。大致逻辑可以为

std::string Util::wildcard = "\\*.*";
void Util::FindAllFolder(const std::string &path, std::vector<std::string> &folders)
{
    WIN32_FIND_DATA wfd;
    std::string newPath = path + wildcard;
    HANDLE handle = FindFirstFile(newPath.c_str(), &wfd);
    if (handle == INVALID_HANDLE_VALUE)
    {
        return;
    }
    while (FindNextFile(handle, &wfd))
    {
        if (std::string(wfd.cFileName) == "." || std::string(wfd.cFileName) == "..")
            continue;
        if (wfd.dwFileAttributes & FILE_ATTRIBUTE_DIRECTORY)
        {
            folders.push_back(path + "\\" + wfd.cFileName);
            printf("\rfind %d folder", folders.size());
            FindAllFolder(path + "\\" + wfd.cFileName,folders);
        }
    }
    FindClose(handle);
}

然后就是根据GID来筛选我们需要的路径。这里又开始我的传统艺能，绕圈子了。因为数据库查出来的GID是字符串，所以我为了查找快一点，先把所有路径的GID截取出来，变成int，存在map里，因为所有GID后都有一个”-“,使用find就可以找到位置，然后substring就好。注意的是，我们一开始存的都是绝对路径，所以find之前还需要substring一下去掉前面的东西，只留文件名，因为我们download的路径是已知的，设为SRC，所以substring(SRC.size())就好。然后循环查找的GID，每次都把字符串变成int来查map。

std::string src ="···\\download";
std::string dest = "···\\book";
Util::FindAllFolder(src, Allfolders);
for (int i = 0; i < Allfolders.size(); ++i)
{
    map[std::stoi(Allfolders[i].substr(src.size() + 1).substr(0, Allfolders[i].substr(src.size() + 1).find('-')))] = Allfolders[i];
    printf("\rdown %d/%dfolder",i+1, Allfolders.size());
}

得到需要的文件夹后，因为WindowsAPI不可以直接复制目录，所以还需要获取目录下文件，和之前获取目录一样，就是变成始文件就添加。循环需要的目录来获取文件。

std::string Util::wildcard = "\\*.*";
void Util::FindAllFile(const std::string &path,std::vector<std::string> &filename)
{
    WIN32_FIND_DATA wfd;
    std::string newPath = path + wildcard;
    HANDLE handle = FindFirstFile(newPath.c_str(),&wfd);
    if (handle == INVALID_HANDLE_VALUE)
    {
        return;
    }
    while (FindNextFile(handle, &wfd))
    {
        if (std::string(wfd.cFileName) == "." || std::string(wfd.cFileName) == "..")
            continue;
        if (wfd.dwFileAttributes & FILE_ATTRIBUTE_DIRECTORY)
        {
            FindAllFile(path+"\\"+ wfd.cFileName,filename);
        }
        else
        {
            filename.push_back(path + "\\" + wfd.cFileName);
        }
    }
    FindClose(handle);
}

拷贝文件

然后，拷贝文件前提是需要文件夹存在，且Windows创建文件夹不能创建多级目录，所以需要先创建一遍label文件夹，然后再创建每个本子的文件夹。

CreateDirectory((dest + "\\" + label[singleLabel]).c_str(), 0);
for (int i = 0; i < folders.size(); ++i)
{
    std::string a = dest + "\\" + label[singleLabel] + folders[i].substr(src.size());
    if(!Util::isDirectoryExist(dest + folders[i].substr(src.size())))
            CreateDirectory((dest +"\\" + label[singleLabel] + folders[i].substr(src.size())).c_str(), 0);
    printf("\rCrestered %d folder", i + 1);
}

最后就是复制文件了，很简单的一个CopyFile函数，有三个参数，第一个源路径，第二个目的地路径，第三个是一个bool值，表示有重复文件存在时候，true表示失败，false表示覆盖。

for (int i = 0; i < files.size(); ++i)
{
    CopyFile(files[i].c_str(), (dest + "\\" + label[singleLabel] + files[i].substr(src.size())).c_str(),false);
